Part 1: "Hello world" and setting up an environment

First of all, let's install Webstorm Javascript IDE as an editor for our project, which is a very convenient tool for developing js applications and has ES6 syntax support. Next step is to install node.js. As I am a windows user, I have installed it from here and nvm (Node version manager) - from here. Note that, first you need to install node and then nvm.

Now, let's create a new blank(empty) project from Webstorm. If we want to check that node.js is correctly installed, run Webstorm terminal (alt + F12) and check with entering a command node -v and nvm -v.

Now, to set up dependecies, we need to run npm init This will ask you some questions, and then write a package.json for you. In the end the package.json looks like this:

{
  "name": "speedtyping",
  "version": "1.0.0",
  "description": "",
  "main": "index.js",
  "dependencies": {},
  "devDependencies": {},
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "author": "",
  "license": "ISC"
}

The assets must be transpiled and bundled by a modern bundler, so we will use Webpack bundler for this. Webpack combines all javascript files and produces(transpiles) them into one (bundle.js) which can be understood by browsers. It also provides a development server to run the code in development environment. This server watches all files and bundles the project again on every change. To install webpack and webpack development server, run following commands: npm install webpack --save and npm install webpack-dev-server --save. After that you will notice that node_modules folder is generated in the project, which consists of all libraries what is needed for webpack. And also you will notice that two lines( "webpack": "^1.12.14", "webpack-dev-server": "^1.14.1") will be added to package.json dependecies. Note that, other developers do not need to run commands for webpack installation again once we have them in package.json, all they do is just npm install and webpack will be installed automatically. To run the code with npm start, we need to add following line to "scripts" in package.json: "start": "webpack-dev-server".

After that we already are able to run a code on the server, which displays just the folders and files we have in a project. Now we are ready to create a simple "Hello world" app. Let's create index.html file with following html code:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Title</title>
</head>
<body>
    Data not from DOM manipulation
    <script type="text/javascript" src="bundle.js"></script>
</body>

</html>

Note that, we do not have bundle.js yet, it will be generated by webpack on the server after transpiling the code. Now we create three javascript files (in js folder), just to have a simple application. First one is hello.js which just manipulates DOM in index.html and puts "Hello world" in the body:

var hello = function(){
    document.body.innerHTML = "Hello World!";
}

module.exports = hello

Second one is robot.js which corresponds the class Robot and has a method which prints some text in a console:

class Robot {
    beep() {
        console.log("beep");
        console.log("boop");
    };
}

module.exports = Robot;

And the last javascript file is the main one (main.js) which requires hello.js file and robot.js files and calls their methods:

var hello = require("./hello.js")
hello()

var Robot = require('./robot');
var robot = new Robot();
robot.beep()

To configure webpack, create webpack.config.js file in root folder of the project and put following code:

module.exports = { entry: './js/main.js', output: { filename: 'bundle.js' }, devServer: { port: 3000 } };

Now create js folder and main.js
