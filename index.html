Part 1: "Hello world" and setting up an environment

First of all, let's install Webstorm Javascript IDE as an editor for our project, which is a very convenient tool for developing js applications and has ES6 syntax support. Next step is to install node.js. As I am a windows user (I am sorry for that :)), I have installed it from here and nvm (Node version manager) - from here. Note that, first you need to install node and then nvm.

Now, let's create a new blank(empty) project from Webstorm. If we want to check that node.js is correctly installed, run Webstorm terminal (alt + F12) and check with entering a command node -v and nvm -v.

Now, to set up dependecies, we need to run npm init This will ask you some questions, and then write a package.json for you. In the end the package.json looks like this:

{
  "name": "speedtyping",
  "version": "1.0.0",
  "description": "",
  "main": "main.js",
  "dependencies": {},
  "devDependencies": {},
  "scripts": {
    "test": "echo \"Error: no test specified\" && exit 1"
  },
  "author": "",
  "license": "ISC"
}

The assets must be bundled by a modern bundler, so we will use Webpack bundler for this. Webpack is a module bundler which combines all javascript files and produces them into one (bundle.js) which can be understood by browsers. It also provides a development server to run the code in development environment. This server watches all files and re-bundles the project on every change.

To install webpack and webpack development server, run following commands: npm install webpack --save and npm install webpack-dev-server --save. After that you will notice that node_modules folder is generated in the project, which consists of all libraries what is needed for webpack. And also you will notice that two lines( "webpack": "^1.12.14", "webpack-dev-server": "^1.14.1") will be added to package.json dependecies. Note that, other developers do not need to run commands for webpack installation again once we have them in package.json, all they do is just npm install and webpack will be installed automatically. To run the code with npm start, we need to add following line to "scripts" in package.json: "start": "webpack-dev-server".

After that we already are able to run a code on the server, which displays just the folders and files we have in a project. Now we are ready to create a simple "Hello world" app. Let's create index.html file with following html code:

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Title</title>
</head>
<body>
    Data not from DOM manipulation
    <script type="text/javascript" src="bundle.js"></script>
</body>

</html>

Note that, we do not have bundle.js yet, it will be generated by webpack on the server after 'bundling' the code. Now we create three javascript files (in js folder), just to have a simple application. First one is hello.js which just manipulates DOM in index.html and puts "Hello world" in the body:

var hello = function(){
    document.body.innerHTML = "Hello World!";
}

module.exports = hello

Second one is robot.js which corresponds the class Robot and has a method for printing some text in a console:

class Robot {
    beep() {
        console.log("beep");
    };
}

module.exports = Robot;

And the last javascript file is the main one (main.js) which requires hello.js file and robot.js files and calls their methods:

var hello = require("./hello.js")
hello()

var Robot = require('./robot');
var robot = new Robot();
robot.beep()

Now we already have javascript files, so we need more 'clever' webpack to run the code and convert it into bundle.js. For that reason, we create webpack.config.js file in root folder of the project and put following piece of code: 

module.exports = {
    entry: './js/main.js',
    output: {
        filename: 'bundle.js'
    },
    devServer: {
        port: 3000
    }
};

With that we are telling webpack that it has to take main.js as an entry and bundle.js as an output file and run the server at 3000 port. Also do not forget to include in package.json that the main file is main.js ("main": "js/main.js").

Run npm start and enter localhost:3000 in the browser. You will see "Hello World" there and "beep" in a console. Note that, the assets are rebundled on change to any of javascript files. E.g put console.log("boop"); in beep function and after refreshing a page boop will appear in a console.

As we are using classes in javascript, it automatically means that we are using es6 (ECMAScript 2015). Since this language is not supported fully in all browsers, we need a transpiler that takes code in ECMAScript 2015 standard and produces it in the older standard that browsers can run. The great tool for that is Babel. To install babel in our project, add following lines in package.json dependencies and then run npm install:

"babel-cli": "^6.5.1",
"babel-preset-es2015": "^6.5.0",
"babel-loader": "^6.2.1",
"babel-preset-react": "^6.3.13",
"babelify": "^7.2.0"

We also need to configure babel loader in webpack.config.js file:

module: {
    loaders: [
        {
            test: /.js?$/,
            loader: 'babel-loader',
            exclude: /node_modules/,
            query: {
                presets: ['es2015', 'react']
            }
        }
    ]
}

After that the project has ES6 support.

Part 2: Simple UI for Speed typing single player game with React components

We will build UI view components using React library. To install it put following dependecies in package.json and run npm install:

"react": "^0.14.7",
"react-dom": "^0.14.7"

A view for a single player game consists of:

  • an area where random words are displayed
  • an input for an user
  • statistics: words per minute and accuracy

The components we are building should be small and simple as possible. We will have some kind of a hierarchy of components and it is important that every component will have only one responsibility. For example, a main Game component does not have to know anything about how to calculate accuracy, this will be the responsibility of the Accuracy component.

Let's first build a component for statistics, which is very simple and will show main idea of composing multiple components in React. We will build 3 components for that: A top-level component which logically is Statistics and two child components, Speed (Words per minute) and Accuracy.

First, create components folder in js to have seperate folder for the React components. Then create
3 javascript files inside it: Statistics.js, Speed.js, and Accuracy.js. Each file corresponds to each component.

Speed component renders a divwhich simply displays a string and value of words per minute:

import React from 'react';
import ReactDOM from 'react-dom';

var SpeedComponent = React.createClass({

    render: function () {
        return (

            <div className="speed">
                Words per minute: {this.props.value}
            </div>
        );
    }
})

export default SpeedComponent;

Similarly, Accuracy component displays div with string and value of accuracy:

import React from 'react';
import ReactDOM from 'react-dom';

var AccuracyComponent = React.createClass({

    render: function () {
        return (
            <div className="accuracy">
                Accuracy: {this.props.value}
            </div>
        );
    }
})

export default AccuracyComponent;

Statistics component renders two components we defined above and passes property values to them. Note that properties are called props in React; They are immutable and are passed from parent component to children.

import React from 'react';
import ReactDOM from 'react-dom';
import SpeedComponent from  './speed'
import AccuracyComponent from './accuracy'


var Statistics = React.createClass({

    render: function () {
        return (
            <div className="statistics">
                <SpeedComponent value={this.props.wpm}/>
                <AccuracyComponent value={this.props.accuracy}/>
            </div>
        );
    }
})

export default Statistics;

As I mentioned above, properties are passed from parents to children, it means that this.props.wpm and this.props.accuracy values will be also passed from parent i.e from the root component. Before we implement the root component let's first create an input component for an user as well (Input.js):

var InputBox = React.createClass({

    getInitialState: function() {
        return {input: ''};
    },

    handleInputText: function (e) {
        this.setState({input: e.target.value});
    },

    render: function () {
        return (
            <div className="inputBox">
                <input
                    type="text"
                    placeholder="Type words here"
                    value={this.state.input}
                    onChange={this.handleInputText}
                />
            </div>
        );
    }
})

export default InputBox;

In this case, we have an additional attribute of the component which is called a state. The html input element property is bound to the state of the input component. On every input change it calls method handleInputTextwhich changes the state of the input.

Now we can display these 2 main components we have already created. Import these two components into main.js file and render them into the main component called GameComponents. 

import InputBox from './components/input';
import Statistics from './components/statistics'

getDefaultProps: function() {
    return {
        wpm: 20,
        accuracy: 10
    };
},   
var GameComponents = React.createClass({
    render: function () {
        return (
            <div className="gameComponents">
                <h1>Speed Typing app</h1>
                <InputBox />
                <br/>
                <Statistics wpm={this.props.wpm} accuracy={this.props.accuracy}/>
            </div>
        );
    }
});

Note that in this case words per minute and accuracy values are not returned by parent component, but they are given from getDefaultProps() function. The result of this function is cached and returned property will have a value if it is not specified by the parent component.

To show the root component and all its children, we need to create main game component and put it into divwith the id called 'content' (Don't forget to add this divin index.html) Here is the snippet to do this:

ReactDOM.render(
    React.createElement(GameComponents, null),
    document.getElementById('content')
);

Now we can already run the app (npm start). You will see the input displayed for an user, where you can type something. You will also see two lines which displays statistics(wpm and accuracy).

The last view we will add is words container. This component takes words array as an input, creates separate word components from them and returns them as a single div as an output:

var WordsContainer = React.createClass({

    render: function () {
        return(
            <div className="words">
                {this.props.words.map((word) => { return <Word key={word} value={word} />; })}
            </div>
        );
    }
})

The word component is separated into Words.js file, which is just responsible to render the word taken from the parent:

var Word = React.createClass({

    render: function () {
        return (
            <div className="word">
                {this.props.value}
            </div>
        );
    }
})

The last step for displaying words is to add random words array in root component and pass it to the WordsContainer component. So we add following line ingetDefaultProps(): words: ["answer", "unfold", "exceptionally", "yourself", "dinners", "struck", "kitchen", "surroundings", "provided"] and following line in render function result: <WordsContainer words={this.props.words} />

After running the project random words are also displayed on the screen. The last thing we need to add is property validations. The validations are done only in development mode and can be seen in browser debugger. An example for a property validadion of the word property is shown below: 

propTypes: {
    value: React.PropTypes.string.isRequired
}

In this case we require String as a type of the word. Similarly we add validations for Accuracy and WPM (numbers) and for a words container (array).
